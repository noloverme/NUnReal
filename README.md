## 🎯 Обзор

**NUnReal** преобразует геймплей Minecraft, предоставляя систему активируемых по чанкам способностей. Игроки могут размещать специальные блоки способностей в чанках, чтобы разблокировать мощные модификаторы, влияющие на:

- Скорость добычи и ломания блоков
- Полёты и передвижение
- Сбор предметов и ресурсов
- Взаимодействие с сущностями
- Рост растений и урожайность
- И многое другое...

### ⚡ Основные возможности

| Функция | Описание |
|---------|----------|
| **15+ Способностей** | От добычи до полёта, от магнита до вечной сытости |
| **27+ Мобов** | Все враждебные и мирные мобы с кастомными рецептами |
| **Система Чанков** | Один блок на чанк с постоянным сохранением данных |
| **Полная конфигурируемость** | Все параметры регулируются через config.yml |
| **Продвинутая обработка событий** | Кастомный крафт, взаимодействие, состояния игрока |
| **Асинхронные задачи** | Магнит предметов и проверка чанков работают без лагов |

## 💻 Установка

### Требования

```
Java 21+
Paper/Spigot Server (протестировано на 1.21)
Maven 3.6+ (для сборки)
```

### Пошаговая установка

#### Шаг 1: Подготовка кода
```bash
git clone <repository-url>
cd NUnReal
```

#### Шаг 2: Сборка проекта
```bash
mvn clean package
```

Скомпилированный JAR появится по пути: `target/NUnReal-1.0.1.jar`

#### Шаг 3: Установка на сервер
```bash
cp target/NUnReal-1.0.1.jar /path/to/server/plugins/
```

#### Шаг 4: Перезагрузка сервера
```
/reload
```

Или просто перезагрузите сервер полностью.

#### Шаг 5: Проверка
```
/nunreal give @s
```

Вы должны получить блок способностей в инвентарь.

## ⚙️ Конфигурация

Плагин полностью настраивается через `config.yml`. При первом запуске автоматически создаётся файл конфигурации со значениями по умолчанию.

### Основные секции конфигурации

#### 1. Крафт (craft)
Определяет рецепт создания блока способностей:

```yaml
craft:
  enabled: true
  slots:
    A: NETHERITE_INGOT    # Левый верхний угол
    B: AMETHYST_SHARD     # Верхний центр
    C: BLAZE_ROD          # Правый верхний угол
    D: DIAMOND            # Центр + боки
  result-name: "§6Блок Способностей"
  result-lore:
    - "§7Поставьте и нажмите ПКМ"
    - "§7чтобы выбрать способность"
```

**Логика:** Рецепт будет выглядеть как:
```
[A] [B] [C]
[D] [D] [D]
[D] [D] [D]
```

#### 2. Параметры блоков (blocks)
Управляет поведением блоков способностей:

```yaml
blocks:
  one-per-chunk: true         # Один блок на чанк
  require-pickaxe: true       # Требует кирку для разрушения
  break-time-ticks: 20        # Время разрушения (20 тиков = 1 сек)
  insta-break: false          # Моментальное разрушение
  drop-item: true             # Выпускает предмет при разрушении
  messages:
    placed: "§aБлок способностей установлен!"
    already-exists: "§cВ этом чанке уже есть блок!"
    broken: "§eБлок способностей разрушен."
    need-pickaxe: "§cИспользуйте кирку!"
```

#### 3. Экономика (economy)
Система стоимости способностей:

```yaml
economy:
  default-cost:
    material: DIAMOND
    amount: 1
  
  ability-costs:
    MINING:           # Копание: 20 алмазов
      material: DIAMOND
      amount: 20
    
    FLIGHT:           # Полёт: 10 незеритовых слитков
      material: NETHERITE_INGOT
      amount: 10
    
    KEEP_RESOURCES:   # Сохранение ресурсов: 32 алмаза
      material: DIAMOND
      amount: 32
    
    # ... остальные способности
```

**Логика стоимости:** Когда игрок активирует способность, из его инвентаря удаляются предметы согласно конфигурации.

#### 4. Меню (menu)
Настройка интерфейса выбора способностей:

```yaml
menu:
  title: "§8Выбор способности"
  size: 18              # Размер инвентаря (должен быть кратен 9)
  filler:
    enabled: true
    material: GRAY_STAINED_GLASS_PANE
    name: " "
  
  abilities:
    MINING:
      slot: 0           # Слот в меню (0-17)
      material: GOLDEN_PICKAXE
      name: "§eКопание"
      lore:
        - "§7+30.0 к скорости ломания"
        - ""
        - "§eЦена: 20 алмазов"
      active-lore-line: "§aАктивно"
```

#### 5. Способности (abilities)
Детальные параметры каждой способности:

```yaml
abilities:
  # Способности на основе атрибутов
  MINING:
    attribute: BLOCK_BREAK_SPEED
    value: 30.0
  
  # Специальные способности
  MAGNET:
    radius: 8.0               # Радиус сбора (блоки)
    speed: 0.30               # Скорость притягивания
    interval-ticks: 5         # Частота проверки
    max-items-per-tick: 100   # Максимум предметов за тик
  
  FLIGHT:
    fly-speed: 0.1            # Скорость полёта
    exit-protection: SLOW_FALLING
    slow-falling-seconds: 5   # Длительность защиты при выходе
```

#### 6. Генераторы мобов (spawners)
Система крафта для спавнеров:

```yaml
spawners:
  enabled: true
  require-silk-touch: true    # Требует "Шёлковое касание"
  
  recipes:
    SKELETON:
      enabled: true
      shape:
        - "G C G"             # Строки рецепта (3x3)
        - "D B D"
        - "A A A"
      ingredients:
        G: "BONE"             # G = BONE
        C: "BOW"              # C = BOW
        B: "NETHERITE_INGOT"  # B = NETHERITE_INGOT
        D: "ARROW"            # D = ARROW
        A: "DIAMOND"          # A = DIAMOND
```

**Логика рецепта:** Верстак будет требовать расстановку предметов в виде:
```
[BONE]        [BOW]         [BONE]
[ARROW]       [NETHERITE]   [ARROW]
[DIAMOND]     [DIAMOND]     [DIAMOND]
```

## 🎮 Команды

### Команды администратора

Все команды требуют права `nunreal.admin` (по умолчанию = OP).

#### `/nunreal reload`
**Описание:** Перезагружает конфигурацию и переинициализирует все рецепты.

```
Использование: /nunreal reload
Результат:
  ✓ Конфиг перезагружен
  ✓ Рецепты способностей обновлены
  ✓ Рецепты спавнеров обновлены
  ✓ Меню переинициализировано
```

**Когда использовать:** После редактирования `config.yml`

#### `/nunreal give [игрок]`
**Описание:** Выдаёт блок способностей игроку. Если игрок не указан, выдаёт себе.

```
Примеры:
  /nunreal give @s              # Себе
  /nunreal give PlayerName      # Конкретному игроку
  /nunreal give @a              # Всем онлайн игрокам
```

#### `/nunreal givespawner <моб> [игрок]`
**Описание:** Выдаёт спавнер определённого моба.

```
Примеры:
  /nunreal givespawner SKELETON
  /nunreal givespawner SKELETON PlayerName
  /nunreal givespawner ZOMBIE @s
```

**Доступные мобы:**
```
Враждебные:    SKELETON, ZOMBIE, WITHER_SKELETON, GHAST, CREEPER,
               HOGLIN, SHULKER, SPIDER, BLAZE, VINDICATOR,
               MAGMA_CUBE, SLIME, ENDERMAN, DROWNED

Мирные:        COW, MOOSHROOM, BREEZE, SNIFFER, IRON_GOLEM,
               ARMADILLO, WOLF, TURTLE, VILLAGER, SALMON,
               COD, CAT, SNOW_GOLEM
```

### Взаимодействие игроков

1. Поместите блок способностей в чанк (ПКМ)
2. Нажмите ПКМ по блоку → откроется меню способностей
3. Кликните на способность в меню
4. Проверьте стоимость и подтвердите активацию

## 🔧 Как это работает

### Логика активации способности

```
1. Крафт → Игрок создаёт блок способностей по рецепту
   ↓
2. Установка → Игрок размещает блок в чанке (ПКМ)
   ↓
3. Взаимодействие → Игрок нажимает ПКМ по блоку
   ↓
4. Меню → Открывается GUI с выбором способностей
   ↓
5. Активация → Игрок выбирает способность
   ↓
6. Проверка → Система проверяет стоимость
   ↓
7. Применение → Если хватает ресурсов:
              - Ресурсы удаляются из инвентаря
              - Способность применяется ко ВСЕМ игрокам в чанке
              - Данные сохраняются
```

### Система сохранения данных

- **Загрузка:** При старте плагина все данные блоков загружаются из памяти
- **Хранение:** Во время работы данные хранятся в памяти (быстро)
- **Сохранение:** При остановке плагина данные записываются на диск
- **Восстановление:** При следующем старте данные восстанавливаются

### Асинхронные задачи

Два фоновых процесса работают отдельно от основного потока:

**ChunkCheckTask** (каждые 20 тиков = 1 сек по умолчанию)
- Проверяет, в каком чанке находится игрок
- Если чанк изменился, переприменяет способности
- Предотвращает "застревание" способностей при движении

**MagnetTask** (каждые 5 тиков по умолчанию)
- Ищет выброшенные предметы в радиусе 8 блоков
- Притягивает их к игроку со скоростью 0.30
- Обрабатывает до 100 предметов за тик (для оптимизации)

## 💡 Примеры использования

### Пример 1: Создание "Зоны добычи"

**Цель:** Создать чанк, где добыча происходит в 2 раза быстрее.

**Конфигурация:**
```yaml
economy:
  ability-costs:
    MINING:
      material: DIAMOND
      amount: 10
```

**Действия:**
```
1. /nunreal give @s          # Получить блок способностей
2. Поместить блок в чанк
3. ПКМ по блоку → выбрать MINING
4. Подтвердить (10 алмазов)
5. Готово! Добыча ускорена в чанке
```

### Пример 2: "Ферма с автоплавкой"

**Цель:** Руда переплавляется сразу в слитки.

**Конфигурация:**
```yaml
abilities:
  AUTO_SMELT:
    affected-blocks:
      IRON_ORE: IRON_INGOT
      GOLD_ORE: GOLD_INGOT
      COPPER_ORE: COPPER_INGOT
    respect-silk-touch: true
    respect-fortune: true
```

**Действия:**
```
1. Активировать способность AUTO_SMELT в чанке (15 💎)
2. Копать руду в чанке
3. Вместо руды падают сразу слитки
4. Экономия времени на плавку
```

### Пример 3: "Зона безопасности"

**Цель:** Создать зону, где враги не спауняются и игрок не падает.

**Конфигурация:**
```yaml
abilities:
  HOLY_GROUND:
    block-raids: false
    blocked-spawn-reasons:
      - NATURAL
      - JOCKEY
      - PATROL
  
  FALL_RESIST:
    attribute: SAFE_FALL_DISTANCE
    value: 10.0
```

**Действия:**
```
1. Активировать HOLY_GROUND (20 💎)
2. Активировать FALL_RESIST (1 💎)
3. В чанке враги не спауняются
4. Можно безопасно падать с высоты
```

### Пример 4: Кастомный спавнер

**Цель:** Создать кастомный рецепт для спавнера скелета.

**Конфигурация:**
```yaml
spawners:
  recipes:
    SKELETON:
      enabled: true
      shape:
        - "B B B"
        - "B S B"
        - "B B B"
      ingredients:
        B: "BONE"
        S: "SPAWNER"
```

**Логика:**
```

[BONE] [BONE] [BONE]

Просто вкладываешь ингредиенты на верстак - получаешь спавнер скелета!
```

## 🔍 Решение проблем

### ❌ Плагин не загружается

**Симптомы:** В логе консоли ошибки при загрузке.

**Решение:**
```
1. Проверить версию Java: java -version
   → Должно быть Java 21 или выше

2. Проверить plugin.yml:
   main: com.noloverme.nunreal.NUnReal

3. Посмотреть логи:
   tail -f logs/latest.log

4. Пересоздать конфиг:
   rm plugins/NUnReal/config.yml
   /reload
```

### ❌ Блок способностей не появляется в крафте

**Симптомы:** Даже с правильными ингредиентами блок не крафтится.

**Решение:**
```yaml
# Проверить в config.yml:
craft:
  enabled: true  # Должно быть true!

# Запустить команду:
/nunreal reload

# Проверить права:
# Игрок должен иметь nunreal.craft
```

### ❌ Способность не активируется

**Симптомы:** Выбираю способность, но эффект не применяется.

**Проверочный список:**
```
□ Достаточно ресурсов в инвентаре?
□ Права nunreal.use установлены?
□ Блок находится в правильном чанке?
□ В чанке уже есть другой блок способностей?
  (система ограничивает: один блок на чанк)
□ Способность включена в config.yml?
```

**Решение:**
```bash
# Перезагрузить конфиг
/nunreal reload

# Снова попробовать
/nunreal give @s

# Если не помогает - проверить логи
tail -f logs/latest.log | grep -i nunreal
```

### ❌ Полёт не работает

**Симптомы:** Активирована способность FLIGHT, но полёт не включается.

**Решение:**
```yaml
# В config.yml проверить:
abilities:
  FLIGHT:
    fly-speed: 0.1  # Не должна быть 0!
    exit-protection: SLOW_FALLING

# Убедиться что:
□ Вы находитесь в чанке с активированной способностью
□ Нижняя часть вашего тела в этом чанке
□ Нет конфликтов с другими плагинами полёта
```

**Отладка:**
```
1. Смена чанка: ПКМ ПКМ ПКМ (нажимайте края чанка)
2. Переход в спектром режим: /gamemode spectator
3. Возврат: /gamemode survival
4. Переподключение: /disconnect → повторное подключение
```

### ❌ Кастомный рецепт спавнера не работает

**Симптомы:** Вкладываю ингредиенты на верстак, но спавнер не создаётся.

**Проверка:**
```yaml
# Убедитесь что:
spawners:
  enabled: true
  recipes:
    SKELETON:
      enabled: true  # true = включен!
      shape:
        - "G C G"
        - "D B D"
        - "A A A"
      ingredients:
        G: "BONE"          # Точные названия!
        C: "BOW"
        B: "NETHERITE_INGOT"
        D: "ARROW"
        A: "DIAMOND"
```

**Проверка ингредиентов:**
```
1. Все ли ингредиенты существуют в Minecraft?
   /give @s BONE 1
   
2. Все ли буквы используются в shape?
   shape:
     - "G C G"   # Используется: G, C
     - "D B D"   # Используется: D, B
     - "A A A"   # Используется: A
   
   Итого используется: G, C, D, B, A ✓

3. Все ли используемые буквы определены в ingredients?
   ingredients:
     G: "BONE" ✓
     C: "BOW" ✓
     D: "ARROW" ✓
     B: "NETHERITE_INGOT" ✓
     A: "DIAMOND" ✓
```

**Решение:**
```bash
# Перезагрузить конфиг
/nunreal reload

# Проверить в логах
tail -f logs/latest.log | grep -i "recipe"

# Выдать спавнер командой
/nunreal givespawner SKELETON
```

## 📈 Производительность

### Оптимизация для больших серверов

**Уменьшить нагрузку ChunkCheckTask:**
```yaml
task:
  interval-ticks: 40  # Вместо 20 (проверка каждые 2 сек)
```

**Оптимизировать магнит:**
```yaml
abilities:
  MAGNET:
    radius: 6.0              # Вместо 8.0
    interval-ticks: 10       # Вместо 5 (проверка реже)
    max-items-per-tick: 50   # Вместо 100
```

**Ускорить разрушение блока:**
```yaml
blocks:
  break-time-ticks: 10  # Вместо 20
```

### Мониторинг TPS

```bash
# Посмотреть TPS прямо в игре
/tps

# Если TPS падает при активации способностей:
1. Отключить асинхронные задачи (временно)
2. Уменьшить radius магнита
3. Увеличить interval-ticks
```

## 📝 Заметки

- Все сообщения в игре на **русском языке** - настраиваются через `config.yml`
- Плагин использует **Bukkit Attribute System** для применения модификаторов
- Обнаружение чанков **автоматическое** через Bukkit API
- Все стоимости **настраиваются отдельно** для каждой способности
- Рецепты спавнеров поддерживают **3×3 сетку** (9 ингредиентов максимум)
- Данные сохраняются **при выключении** плагина и **загружаются при включении**
